# INS-001: 全局 MAS 体系的三足鼎立架构

> **状态**: 🟢 ACTIVE
> **日期**: 2026-02-26
> **所属模块**: TA (Meta-repo) 全局专栏

## 1. 架构动机

原有的 `CTS1` 等单体系统将数据获取、回测引擎、交易逻辑与执行端口深度耦合。随着 MAS（多智能体系统）需求的发展，为了彻底打破“环境硬编码”的瓶颈并拥抱大语言模型的 Tool-use 生态，系统演进为三个物理与逻辑完全解耦的基础设施模块。

## 2. 核心模块与职能边界

### 2.1 TradingAgentArena (TAA) - “虚拟环境与物理法则”
- **角色**：承载真实与模拟竞技的底座服务。
- **定位**：一个持续运行的高性能后台服务群。
- **生态接入**：它将自身包装为一种环境服务，通过提供特定的鉴权与入口向任意种类的 Agent 提供“参赛准入许可”。即便是完全外部架构（如 OpenClaw）依然可以通过该通道参与收益博弈。

### 2.2 TradingAgentSquad (TAS) - “认知大脑与玩家代表”
- **角色**：具备思考、沟通与规划能力的交易团队。
- **定位**：基于 AAAgent 构建的实战主体。
- **核心交互**：团队成员拥有各自独立的认知（Skills），它主动订阅感知环境（TAA）的影响，并负责做出主观发单决策。

### 2.3 TradingSkills (TS) - “工具箱与兵工厂”
- **角色**：将具体的交易动作与复杂的数学策略静态封装的能力模块。
- **定位**：严格遵守国际化 Skills 架构原则（而非动态的 MCP 服务端）的代码集合，分为两大基础类型：
  - **策略型 (Strategy Skills)**：提供纯逻辑计算，不涉及底层联网。
  - **执行型 (Execution Skills)**：作为类“ERC 标准”，向整个应用体系承兑买卖下单等实际操盘动作。

## 3. 协作闭环

通过隔离执行环境 (TAA)、独立智能体 (TAS) 以及标准化能力组件 (TS)，本架构彻底分离了“比赛场”、“选手”与“武器”的边界，实现了加密世界的分布式协作理想。
