# INS-001: 交易智能体体系架构演进与 Skill 标准化

> **状态**: 🟢 ACTIVE
> **日期**: 2026-02-26
> **作者**: VibeCoding 协作组

## 1. 背景与挑战

原有的 `CTS1` 等单体系统将数据获取、回测引擎、交易逻辑（如 Grid RSI）与执行端口深度耦合。随着 MAS（多智能体系统）需求的发展，单一智能体（Agent）面临以下挑战：
1. **策略冗余**：如果有 10 个策略，底层执行和回测代码会被复制 10 次。
2. **大模型适配痛点**：传统的 API 或函数对大语言模型（LLM）不透明，Agent 难以直接将其视为一种“即插即用”的工具。
3. **环境硬编码**：回测（模拟盘）与真实交易所（实盘）难以由 Agent 根据状态上下文动态切换。

## 2. 三足鼎立的顶层架构 (TAA/TAS/TS)

为了实现彻底的职责剥离，系统被拆分为三大完全独立的物理层级与 Git 仓库：

### 2.1 TAA (Trading Agent Arena) - “角斗场”基建
- **定位**：一个持续运行的高性能后台服务群。
- **职责**：
  - 提供数据泵（Tick/Kline 流转）。
  - 创建虚拟时钟（Virtual Clock）以支持极速的回测竞技。
  - 接管严谨的撮合引擎与绩效统计（榜单、夏普、回撤）。
  - **无需关心策略**，它只是一片客观的战场。

### 2.2 TAS (Trading Agent Squad) - “大脑指挥部”
- **定位**：基于 AAAgent 构建的玩家实体本身。
- **职责**：
  - 管理诸如 PA（个人助手）、Tech Analyst 等 Agent 角色的认知、记忆与多方辩论（Debate/Consensus）。
  - 它是整个交易系统的“意志”。它观察客观表象（来自 TAA），并挥舞武器（来自 TS），发布最终的动作指令。

### 2.3 TS (Trading Skills) - “兵工厂”
- **定位**：遵循 Anthropic Skills/MCP 标准的工具包集散地。
- **本质**：任何包含 `SKILL.md`（给 LLM 看的系统提示词手册）和具体执行脚本的文件夹。Agent 学会该 Skill，就获得了对应的能力。

---

## 3. TS 生态：被释放的“核威慑”

在讨论中，最关键的突破在于对 **Skill 是什么** 的认知矫正：

### 3.1 让 Agent 作为“包工头”拉起常驻服务
针对高频（如网格）策略，它们不应该由大模型通过慢速的 MCP API 去逐根 K 线交互。
相反，Agent 通过调用 `Grid RSI Skill` 提供的指令，**在它的本地环境（如沙盒电脑中）直接运行 Python 脚本拉起一个低延迟守护进程（Daemon）**。从而实现了“大模型定战略，本地机器打战术”的完美分工。

### 3.2 `trading_exe`：成为交易界的“ERC 标准”
为了解决策略间的底层代码冗余，TS 必须内含一个底层构建设施包：`trading_exe` Skill。

*   **这是什么？** 它包含了所有的交易所对接网关（Executors）、数据拉取逻辑（Datafeeds）与订单薄模型。
*   **如何工作？**  所有的策略包（如 `grid_rsi_4.0_Jeff_Huang`）不再自己造轮子连接 OKX。它们在其脚本内直接 `import trading_exe` 来完成最终的动作。
*   **深远影响**：
    1.  **极度解耦**：当 OKX 更改了鉴权 API，我们只需修改 `trading_exe`，成百上千的上层策略瞬间恢复健康。
    2.  **一键切环境**：Agent 只需要告诉 `trading_exe` 是连向“TAA 竞技场”还是“OKX 真实 API”，上层策略无需修改一行代码。
    3.  **脱离策略的纯净交易**：一个足够聪明的 Agent，即使不加载任何具体的算法策略，只要它读懂了 `trading_exe` 的说明书，就能直接凭自己的判断下发市价买单。赋予了系统最高的“副驾驶（Copilot）”自由度。

## 4. 结论与下一步

将通信面（大模型读懂的 Skill 协议）与执行面（极速的 `trading_exe` 底层进程）相分离，是实现长效、稳健且充满想象力的 Crypto MAS 平台的基石。
接下来的核心工程任务，将是先把 `CTS1` 中经过了实弹验证的引擎组件抽取至 `trading_exe`，随后以此标准重铸第一个策略 Skill。
