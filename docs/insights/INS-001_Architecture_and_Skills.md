# INS-001: Trading Skills 设计思想与分类规范

> **状态**: 🟢 ACTIVE
> **日期**: 2026-02-26
> **作者**: VibeCoding 协作组

## 1. Trading Skills 设计理念

Trading Skills (TS) 是为加密交易场景专门设计的标准化能力工具箱。它严格遵循开放的大语言模型技能协议（如 Anthropic Skills / MCP）。
在 TS 的设计哲学中，技能不再是硬编码在单体程序内部的函数，而是独立的、可即插即用的资源包。每个 Skill 包含一个向 AI 解释使用说明的 `SKILL.md`（系统提示词），以及用于底层运转的执行脚本。通过此设计，多智能体系统获得了类似“生态配件”的扩展能力。

## 2. Skill 的双轨分类模型

为了实现极致的架构解耦与防冗余（DRY），TS 内部将技能从逻辑上严格划分为两类：**交易执行型 (Execution Skills)** 与 **策略型 (Strategy Skills)**。

### 2.1 交易执行型 Skill (Execution Skills)
此类 Skill 是系统的技术基建，直接对接外部数据源（如交易所 WebSocket）与订单网关。
- **代表组件**：`trading_exe`。
- **定位与价值**：它类似于区块链领域的“ERC 接口标准”。它向上层提供统一的接口（如：查询资金、获取 Tick、挂单、撤单），而隐藏了底层的 API 鉴权、签名与连接维持逻辑。
- **应用场景**：
  - 作为其他所有“策略型 Skill”的共享依赖底座。
  - 特别地，任何具备决策能力的 Agent，即使不加载任何专业算法策略，只要它理解了 `trading_exe` 的说明书，就能直接凭自身的逻辑推理自主下达买卖指令（Copilot 及高阶自决模式）。

### 2.2 策略型 Skill (Strategy Skills)
此类 Skill 是纯粹的交易逻辑资产，负责基于输入数据生成买卖信号或触发高频机器人，它们**绝对不直接调用任何交易所 API**，而是全部通过调用 `trading_exe` 完成动作。
- **代表组件**：`grid_rsi_4.0_Jeff_Huang`。
- **工作模式**：
  - **低频模式（趋势计算）**：Agent 在特定周期（如每小时）调用该 Skill 提供的脚本，脚本瞬间处理 K 线并返回结构化研判信号供 Agent 决策。
  - **高频/托管模式（守护进程）**：针对网格等需秒级响应的策略，Agent 调用该 Skill，实质是在本地环境中拉起一个低延迟且无需模型干预的 Python 后台子进程。Agent 仅负责“发号施令（部署配置/启停）”及监控状态。

## 3. 与外部系统的交互

- **TAA (Trading Agent Arena)**: 竞技场平台可以提供一个专门的 `TradeArena Skill`。外部 Agent（不论其底层架构架构）只要加载了该 Skill，就获得了进入 TAA 回测或模拟比赛的入场券和标准交互协议。
- **TAS (Trading Agent Squad)**: TAS 的团队成员（如分析员、PA 队长）可以根据自身的分析权重偏好，自由订阅和组装上述的不同 Skill 来强化自身的实战能力。

## 4. 结论

将策略智慧从底层执行器中剥离，把 `trading_exe` 确立为公共基础协议，是构建一个繁荣、安全且可进化的加密智能体生态的关键步骤。这种声明式（Agent）与执行式（Scripts）相分离的做法，最大化了多智能体系统在加密交易领域的长期生命力和社区可维护性。
